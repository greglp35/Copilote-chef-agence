FleetOS — Migration Supabase complète

Document de référence architecture cible.

Périmètre Auth / Contexte utilisateur

  • Authentification via Supabase Auth (session navigateur) — source de vérité principale.
  • Chargement du profile utilisateur (incluant current_agency_id).
  • Chargement des memberships actifs.
  • Résolution de l'agence active avec priorité: profile.current_agency_id → préférence locale temporaire → premier membership actif.
  • Fallback legacy temporaire autorisé uniquement pour éviter de casser les écrans non migrés.

Contraintes sécurité

  • RLS obligatoire sur toutes les tables métier exposées au front.
  • Aucun accès cross-agence hors politiques explicites.
  • shared_tournee_feed et interagency_slot_requests gérés par politiques dédiées.
  • Le front ne doit jamais faire confiance à des rôles/permissions purement locaux.

Modèle de contexte front

{
  session,
  profile: { id, email, current_agency_id, ... },
  memberships: [{ agency_id, role, status }],
  activeAgencyId,
  permissions
}
